# Decision Journal: 2026-04-18

## Decisions

- **WL-115 default_locale CHECK 제약 확장 완료** — 화이트리스트(ko/en)를 관계적 정합성(`default_locale = ANY(published_locales)`)로 교체. 이유: ko/en/ja/zh + 확장 로케일의 사업 계획을 DB 레벨에서 지원하기 위해. Forward-only 롤백 정책 채택 (Option A).
- **WL-116 AI 개발 공정 4대 약점 보완 적용** — Drift Precheck + Shadow Migration Check + Adversarial Auditor 프롬프트 + `docs/tech-debt.md` 체계 + husky pre-commit hook. 이유: WL-105/110 실패 패턴이 "규칙은 있으나 기계적 강제가 없다"였음. 자기 규율에 의존하는 프로세스는 피곤한 날에 깨진다.
- **WL-118 Design 완료 — Triangular Assertion + `setSession()` 단일화** — RLS 자동 검증 구조를 positive-control / negative-control / target 3단으로 설계. 이유: Auditor R-LF-1 지적 "0건 반환만으로는 RLS 차단과 데이터 부재 구분 불가". 헤더 주입 fallback도 완전 제거 (auth.uid NULL fallback 위험).
- **`docs/journal/` 체계 도입** — 비개발자 오너의 이해도 부채를 줄이는 유일한 실질 수단. 이유: Jira/audits/tech-debt가 전부 "결과물"만 기록. 티켓 사이의 연결, 기각한 경로, AI와의 합의/갈등은 어디에도 안 남는다.

## Rejected Paths

- **Multi-model review (GPT-4o/Gemini 교차 검토)** — AP의 첫 제안. 기각 이유: AP Architect가 이미 Claude와 다른 모델인데 to_jsonb 이슈·LaTeX 수식에서 두 번 틀렸다. 원인은 "같은 모델 계열의 확증 편향"이 아니라 **컨텍스트 부재**. 모델을 바꿔도 WL-83 JSONB 이력을 모르면 똑같이 틀린다. → 대신 Auditor 프롬프트에 grep 선행 단계 추가.
- **`supabase db reset` 필수화** — AP의 두 번째 제안. 기각 이유: 이 프로젝트의 seed.sql은 partner·contents·RLS 설정까지 포함하여 재실행 비용이 높다. 데이터 변환 마이그레이션에만 선택 적용이 실용적.
- **LaTeX 기반 Migration Cost 공식 강제** — AP 제안. 기각 이유: 스타트업 MVP 규모에서 ALTER TABLE lock은 실질 위험 아님. 예쁜 수식이 실행 결정에 영향을 줘서는 안 된다.
- **Vitest Projects 분리 (보안 테스트 별도 프로젝트)** — AP 제안. 기각 이유: Projects 설정 복잡도 > 실행 속도 이득. ~50개 테스트로 전체 2~3분 증가 수준. 필요 시 `npm run test:rls` 별도 script로 분리 가능.
- **저널 작성을 "다음 세션 첫 Task"로 미룸** — AP 제안. 기각 이유: 오늘 세션의 의사결정 밀도가 매우 높아 하루만 지나도 정확도가 떨어진다. **지금 바로** 작성이 타이밍상 최선.
- **기존 `partner-admin-e2e@test.local`을 `partner-a-admin-e2e`로 리네임** — 대칭 네이밍을 위해 검토. 기각 이유: `auth.setup.ts`·`.auth/partner.json` storageState·기존 Admin E2E 다수가 하드코딩 참조. 리네임 회귀 위험 > 비대칭 심미적 불편.
