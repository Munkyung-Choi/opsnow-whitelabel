# Decision Journal: 2026-04-23

## Decisions

- **`extractI18n` 빈 i18n 객체 처리 분기 도입** — `{"ko":"","en":""}` 같이 i18n 키는 있으나 모든 값이 빈 문자열인 TEXT 컬럼이 UI에 raw JSON 문자열로 노출되던 경로를 null 반환으로 차단(`4ca7580`). 비-i18n JSON 객체(`{"url":"..."}`)는 원본 유지하여 backward compatible. 이유: eval-arch Testability B 검증 중 부수 발견, 섹션 파서 null-safe 폴백이 이미 있어 null이 더 안전한 default.
- **WL-156 회고의 코드화 — Premise Gate + 영향 파일 이력 검증** (`529e154`) — 단순 회고 문서에서 끝내지 않고 CLAUDE.md High/Critical 트랙 Context 단계에 강제 절차로 삽입. 이유: WL-156에서 "proxy.ts 비활성" 전제 오류 후 피벗한 실손실을 재발 방지. 같은 맥락에서 aisend 스킬에도 3라운드 상한 명시(5차 결함 → 6차 "견고" 뒤집기 실증).
- **Atomic Modification 분리 커밋 준수** — extractI18n fix · 테스트 · 거버넌스 업데이트를 서로 다른 관심사로 판별해 5개 커밋으로 분리 push. 이유: 롤백 단위를 최소화, 기능 변경과 문서 거버넌스가 같은 커밋에 섞이지 않도록.

## Rejected Paths

- **submitLead·extractI18n fix에 Jira 티켓 신규 발급** — 기각. eval-arch 보고서 항목으로 추적성 확보, 3줄 fix에 티켓 overhead 과함. 커밋 메시지에 `eval-arch Error Handling B`, `Testability B` 출처 명시.
- **Playwright 대량 실패 시 즉시 `extractI18n` fix를 revert하여 회귀 격리** — 기각. 타임아웃 정확히 5.8초(expect 기본 타임아웃)라는 패턴이 렌더링 실패를 시사했고, 직접 curl로 확인 시 내가 띄운 dev(3001)는 200 정상 응답, 포트 3000의 좀비 프로세스만 500. 코드 revert 대신 **실체 확인 우선** 진단이 더 빠름.
- **`reuseExistingServer=false`로 Playwright config 강경화** — 보류. 로컬 첫 실행 시 dev 서버 재기동으로 warm-up 지연 트레이드오프 발생. 오늘은 "좀비 프로세스 주의"를 운영 지식으로 남기는 선에서 마무리.

## Debts & Risks

- **좀비 dev 서버 관리 부채** — Playwright `webServer.reuseExistingServer`가 깨진 기존 서버(HTTP 500)를 재사용하면 전수 실패가 "테스트 문제"로 오해된다. 오늘 첫 run에서 `9 passed / 59 did not run / 16분` 결과가 나온 직접 원인. 운영 체크포인트: 이상한 테스트 실패 시 `curl http://partner-a.localhost:3000/` 로 200 여부를 **가장 먼저** 확인.
- **CI 최근 2개 failure 미해결** — 직전 `d4b0c76` (parsers.ts 분리), `191b2ac` (observability env Zod) 2연속 실패. 오늘 push한 `529e154`는 in_progress. 이번 세션에서는 원인까지 내려가지 못했다.
- **`extractI18n` 비-i18n JSON 객체 통과 경로** — 현재 fix는 i18n 키(ko/en/ja/zh) 중 최소 1개 string이 있어야 i18n 객체로 판정. 즉 `{"url":"..."}` 같은 구조 JSON은 여전히 raw 문자열로 통과한다. 섹션 파서들이 이 경로를 실제로 소비하는지 장기적으로 스캔 필요(오늘 단위 테스트에만 가드됨).
- **Admin logo upload 1 flaky** — Playwright 오늘 run 결과. 이번 변경과 무관하지만 Storage 관련 간헐 실패 누적 중.
