## Decisions

- **Origin Sync Gate 신설 (하네스 보완)** — 4일·62커밋 미push 사태(Context Silo)로 CI가 오래된 코드를 검증하는 구조적 문제가 발생했음. `git log origin/main..HEAD` 가드를 `/dev-start` STEP 2에 추가하고 CLAUDE.md에 명문화. unpushed ≥ 10 또는 24시간 초과 시 `[SYNC_REQUIRED]` 플래그 발동.

- **uploadPartnerAsset throw → Result 타입 전환** — Storage 업로드 실패 시 고아 파일이 남는 문제를 발견. `throw` 대신 `{ ok, error }` Result 패턴으로 전환하고 실패 시 rollback 로직 추가. WL-132 테스트도 새 API 시그니처에 동기화.

- **WL-134 섹션별 discriminated union 스키마** — 섹션 타입별 페이로드를 `any`로 처리하던 구조를 discriminated union으로 교체. 잘못된 섹션 타입 페이로드를 컴파일 타임에 차단하여 런타임 오류 원천 제거.

- **WL-42 ContactForm → leads DB 저장 완료** — Server Action 연동으로 마케팅 리드 수집 루프 폐쇄. DEBT-002(ContactFormFields 로직 중복)도 refactor 커밋으로 동시 상환.

- **WL-141 GoTrue deleteUser 제거** — E2E purgeTestUsers에서 `deleteUser` 사용 시 GoTrue 소프트 삭제 트랩에 빠져 재실행마다 중복 유저 충돌 발생. `deleteUser` 제거 후 DB 직접 정리 패턴으로 전환하고 cleanup_e2e_test_users RPC 신설(04-21 오전 커밋).

- **/aisend 스킬 신설** — 외부 LLM 피드백을 코드베이스와 교차검증하는 공식 채널 확보. 외부 피드백이 구현 지시로 오인되는 것을 구조적으로 차단.

- **ESLint §9.3 도메인 경계 자동화** — `no-restricted-imports`로 marketing↔admin 컴포넌트 교차 import를 빌드 시 자동 차단.
