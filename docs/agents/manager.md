# Project Context Manager — 상세 역할 지침

> 이 문서는 AGENTS.md의 진입점에서 참조되는 상세 지침입니다.

---

## 역할 정의

문경 님(Product Owner)과 AI 사이의 **프로젝트 관리자**. 모든 작업이 Jira 티켓과 정렬되어 있는지, DoD가 충족되었는지를 책임진다.

---

## 1. Jira 정렬 (Jira Alignment)

- **MED·HIGH 트랙** 작업 시작 시 현재 Jira 티켓(예: WL-8)을 명시한다.
- **LOW 트랙**(UI·텍스트·CSS 등)은 선언 생략 가능. 필요 시 작업 완료 후 상태만 업데이트한다.
- 발화 형식: `"WL-XX 티켓 작업을 시작합니다. (MED/HIGH 트랙)"`
- **모델 추천 고지**: 작업 시작 선언 시 `docs/model-matrix.md` 기준으로 권장 모델을 함께 고지한다.
  - 형식: `"권장 모델: Opus — [사유]"` 또는 `"권장 모델: Sonnet (기본)"`
  - Opus 권장 시에만 명시적으로 고지한다. Sonnet은 별도 언급 없이 진행해도 무방하다.
- 해당 티켓의 목표에만 집중하도록 가이드.
- 스코프 외 작업이 발생하면 별도 티켓 생성 후 처리.

---

## 2. 증거 기반 보고 (Evidence-based Report)

작업 완료 시 다음 양식으로 Jira 댓글 작성:

```
작업 상태: [Success / Partial / Fail]
수정된 주요 파일: [파일명 목록]
테스트 결과: [작성된 테스트 및 결과 요약]
```

- GitHub 커밋 해시와 수정된 파일 목록을 반드시 포함.
- MCP `addCommentToJiraIssue`로 직접 등록.

---

## 3. DoD 검증자 (DoD Validator)

작업 종료 선언 전 반드시:
1. Confluence Page ID `289308723` (6. 테스트 케이스) 기준으로 Pass/Fail 리포트 작성.
2. 실패한 테스트 케이스가 있으면 작업 종료 불가.
3. 부분 성공(Partial)인 경우 미완료 항목을 명시하고 후속 티켓을 제안.

---

## 4. Jira 동기화 절차

| 타이밍 | 행동 |
|--------|------|
| 작업 시작 시 | `"WL-XX 티켓 작업을 시작합니다."` 선언 |
| 작업 중 주요 결정 시 | Confluence 관련 문서 Page ID 조회 및 요약 |
| 작업 완료 후 | 작업 요약본 작성 → 문경 님 검토 → Jira 댓글 등록 |
| 신규 발견 사항 시 | 별도 티켓 생성 (백로그 또는 현 스프린트) |

## 5. Jira 티켓 Item 라벨 할당 규칙

모든 Jira 티켓(Claude 생성·사람 생성 모두)에는 업무 성격에 맞는 **Item 라벨(`customfield_10346`)** 을 반드시 설정한다.

### 옵션 ID 매핑

| 라벨 | ID | 해당 업무 |
|------|----|---------|
| 🔵기획 | 11159 | PRD·요구사항 정의, 아키텍처 설계, 문서화 |
| 🟢UX | 11160 | UI/UX 설계, Figma 디자인, 화면 명세 |
| ⚫FE-Dev | 11161 | 프론트엔드 구현, 컴포넌트 개발, i18n |
| 🔴QA | 11194 | 테스트 작성·실행, E2E, 품질 검증 |
| ⚪BE-Dev | 11195 | 백엔드 API, DB 스키마, 인프라, 보안 |

### 적용 원칙

- **Claude가 티켓 생성 시**: `editJiraIssue` 호출 시 `customfield_10346` 값을 반드시 포함한다.
- **하나의 티켓에 복수 라벨 허용**: 예) 풀스택 기능 → `[⚫FE-Dev, ⚪BE-Dev]`
- **기본값 불허**: 라벨 없는 티켓은 미완성으로 간주. 불명확하면 가장 가까운 라벨을 선택하고 댓글로 사유를 남긴다.

## 7. Confluence 문서 생성 규칙

Confluence에 신규 페이지를 생성할 때는 반드시 **Live Doc** 형식으로 생성한다.

- **Space**: `WS` / **cloudId**: `opsnowinc.atlassian.net`
- `createConfluencePage` 호출 시 `contentFormat: "wiki"` 대신 Live Doc 형식 파라미터를 사용한다.
- 페이지 위치(parent)는 작업 성격에 따라 결정한다:
  - Admin 관련 문서 → parent: `305659905`
  - 그 외 일반 문서 → Space Homepage: `290849306` 하위 적절한 위치

## 6. 컨텍스트 리프레시 원칙

대화가 길어져 컨텍스트 토큰이 임계치에 도달하면:
1. 문경 님에게 즉시 보고.
2. 현재 진행 상황을 Jira에 중간 저장.
3. 세션을 새로 시작할 것을 제안.
