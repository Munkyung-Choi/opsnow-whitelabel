## 📝 Auditor Digest

### Core Logic
`scripts/gen-types.sh`는 Supabase CLI를 래핑하여 타입 재생성 + 성공 메시지를 출력한다.
`scripts/check-env.sh`는 `.env.local`을 파싱하여 필수 4개 변수의 누락(키 없음)과 빈값을 구분하여 보고한다.
`.gitattributes`로 `.sh` 파일의 LF 줄바꿈을 Git 레벨에서 강제한다.

### Potential Risks

1. **Windows CRLF 오염**: `.gitattributes` 설정 전 커밋된 `.sh` 파일이 CRLF로 저장되어 있을 경우 `bash: /r: command not found` 오류 발생 가능. → `.gitattributes` 적용 후 기존 파일은 `git add --renormalize .`로 재정규화 필요 (이번 커밋에서 신규 생성이므로 해당 없음).
2. **RESEND_API_KEY 강제 검사**: 이메일 없이 로컬 개발하는 환경(예: 리드 API를 테스트하지 않는 경우)에서 check-env.sh가 항상 실패. → 향후 `--skip` 플래그 또는 선택적 변수 분리를 고려할 수 있음.
3. **gen-types.sh의 Supabase CLI 의존성**: `npx supabase`가 없는 환경(CI 등)에서 실패. → CI에서는 `npm run gen:types`를 실행하지 않으므로 현재는 문제 없음. 단, 로컬 Supabase CLI 미설치 시 명확한 에러 메시지 없이 실패할 수 있음.

### Questions for Reviewer
1. `check-env.sh` 필수 변수 목록이 실제 `.env.local` 요구사항과 일치하는가? (특히 `RESEND_API_KEY`가 현재 단계에서 필수인지)
2. `gen-types.sh`에 Supabase CLI 설치 여부 사전 체크(`command -v supabase`)를 추가해야 하는가?
3. Husky pre-commit에 `check:env`를 연결할 시점은 언제가 적절한가? (현재는 수동 실행)

### Resolution
- Risk 1: 신규 파일 생성이므로 CRLF 오염 없음. `.gitattributes` 적용으로 이후 커밋부터 LF 강제.
- Risk 2: 현재 단계에서 RESEND_API_KEY는 필수로 유지. 개발 편의를 위한 분리는 SKILL.md에 TODO로 기록.
- Risk 3: CI에서 gen-types는 실행하지 않으므로 현재 허용 범위.
- Q3: Husky 연결은 `.env.local`이 팀 공유 환경에서 표준화된 이후로 지연. 현재는 SKILL.md 가이드로 대체.
