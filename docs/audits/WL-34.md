## 📝 Auditor Digest

### Core Logic
`scripts/gen-types.sh`: Supabase CLI를 래핑하여 타입 재생성 + 파일 경로 + 타임스탬프를 출력한다.
`scripts/check-env.sh`: `.env.local`을 파싱하여 누락(키 없음)과 빈값을 구분 보고하고, `.env.example` 참조 안내 및 Supabase 대시보드 링크를 출력한다.
`.gitattributes`로 `.sh` 파일 LF 강제. `git update-index --chmod=+x`로 Git 레벨 실행 권한(`100755`) 부여.

### Potential Risks

1. **Windows CRLF 오염**: `.gitattributes` 설정으로 신규 파일은 LF 강제됨. 기존 커밋된 파일은 `git add --renormalize .`로 재정규화 필요 시 적용.
2. **RESEND_API_KEY 강제 검사 마찰**: `npm run dev` 선행 연결(hard-block) 미구현 — 아래 설계 결정 참조.
3. **gen-types.sh Supabase CLI 의존성**: 로컬 미설치 시 명확한 에러 없이 실패 가능. 향후 `command -v supabase` 사전 체크 추가 권고.

### 설계 결정 기록

**`npm run dev` + `check:env` 연결 — 비연결로 결정**

- **검토 결과**: DoD가 "검토"로 명시함 (강제 구현 아님).
- **결정 사유**: 현재 `.env.local`에 `RESEND_API_KEY`가 없는 상태. 하드 블록 연결 시 `npm run dev` 즉시 실행 불가 — 개발 마찰 발생.
- **대안**: 개발자는 세션 시작 시 `npm run check:env`를 명시적으로 실행. SKILL.md에 가이드 추가.
- **재검토 시점**: `RESEND_API_KEY`를 포함한 모든 필수 환경변수가 `.env.local`에 확인된 이후 `"predev": "npm run check:env"` 추가 권고.

### Questions for Reviewer
1. `RESEND_API_KEY`가 현재 단계에서 필수인지, 아니면 선택적으로 분리할 수 있는지 확인 필요.
2. `npm run dev` 하드 블록 연결 시점에 동의하는가?

### Resolution
- Q1: RESEND_API_KEY는 리드 알림 이메일 기능 구현 단계까지 선택적으로 처리 검토 가능.
- Q2: 모든 필수 환경변수 확인 후 `predev` 연결 예정. Gemini 본부장 감사 의견 반영.
