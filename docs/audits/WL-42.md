# Auditor Digest — WL-42 Contact Form Server Action

> **작성일**: 2026-04-20
> **트랙**: High (🟨)
> **감사자**: Security Auditor sub-agent

---

### Core Logic

- `submitLead(formData)` Server Action: `host` 헤더 → 서브도메인 파싱 → anon DB 조회로 `partner_id` 서버 resolve
- Zod 검증 후 anon 클라이언트 + `leads_public_insert` RLS(`is_active=true` 파트너만 허용)로 INSERT
- `system_logs` 감사 로그 생략 — `actor_id UUID NOT NULL REFERENCES auth.users(id)` FK 제약으로 구조적 불가. `leads` row 자체가 감사 증적.

---

### Identified Risks

| # | 등급 | 위험 | 수정 지시 |
|---|------|------|----------|
| 1 | **HIGH** | hidden `partner_id` FormData 필드를 Server Action이 신뢰 시 크로스테넌트 lead INSERT 가능. RLS는 `is_active=true`만 검증하므로 DB 레벨 차단 없음. | `formData.get('partner_id')` 절대 사용 금지. `host` 헤더 resolve 경로만 사용. Scope B(hidden 필드 제거) 반드시 이행. |
| 2 | **HIGH** | host resolve 실패 케이스 미처리. DEV_PARTNER_SLUG(plain localhost), Vercel Preview URL(`*.vercel.app`), Custom Domain, null host 케이스에서 resolve 실패 → INSERT 시도 또는 기능 전체 불동작. | `resolve-partner-from-host.ts`가 `proxy.ts`의 `resolvePartnerFromHost()` 분기(DEV fallback + subdomain + custom_domain + null 처리)와 동일한 로직을 따라야 함. |
| 3 | **LOW** | 허니팟(`company_website`)이 클라이언트에서만 검증. 직접 fetch로 우회 가능. DEBT-005(Rate Limiting 미구현)와 결합 시 스팸 1차 방어 무력화. | Server Action 내 `formData.get('company_website')` 비어있지 않으면 silent success 반환(DB 미접근). |
| 4 | **MEDIUM** | DEBT-002 미상환 시 Server Action 코드가 두 컴포넌트에 복붙되어 보안 로직 SSOT 위반 영속화. | WL-42 Impl 착수 전 `ContactFormFields` 서브컴포넌트 또는 `useLeadSubmit` 훅 추출 후 DEBT-002 상환. |

### Human Check 승인 포인트

1. **system_logs audit 생략(옵션 A)** — `leads` row 자체가 감사 증적으로 인정되는지 확인
2. **SSOT 중복 허용** — `proxy.ts`(Edge) / `resolve-partner-from-host.ts`(Node) 서브도메인 로직 중복을 허용된 부채로 간주, DEBT-006 등록
3. **DEV_PARTNER_SLUG 폴백 포함 여부** — `resolve-partner-from-host.ts`에서 `process.env.DEV_PARTNER_SLUG`를 plain localhost 케이스 폴백으로 사용할지 확인
