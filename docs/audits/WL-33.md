## 📝 Auditor Digest

### Core Logic
`app/not-found.tsx`를 Next.js 표준 not-found 컴포넌트로 구현. `export const metadata`로 "404 - Page Not Found" 타이틀 설정.
`bg-background`, `text-foreground`, `bg-primary` 등 CSS Variable 기반 Tailwind 클래스로 브랜딩 시스템에 종속.
`<Link href="/">` 사용으로 현재 호스트(도메인)를 유지하며 루트로 이동.

### Potential Risks

1. **shadcn Button 미사용**: `src/components/ui/button`이 없어 Link에 직접 Tailwind 클래스 적용. 향후 Button 컴포넌트 추가 시 일관성을 위해 교체 필요.
2. **metadata와 not-found의 타이틀 충돌 가능성**: 상위 layout.tsx의 기본 metadata title이 이 페이지의 title을 덮어쓰지 않는지 확인 필요. Next.js는 가장 하위 metadata를 우선하므로 현재는 문제없음.
3. **파트너 브랜딩 미적용**: CSS Variable이 파트너별 테마 주입 전(`[partnerId]/layout.tsx`) 단계에서 호출되므로 OpsNow 기본 테마로만 표시됨. 파트너 도메인에서 직접 UUID 접근 시 파트너 색상이 아닌 기본 색상 표시 — 설계 의도에 부합.

### Questions for Reviewer
1. DoD에 명시된 audit 파일명이 `WL-35.md`인데 티켓은 WL-33임. `WL-33.md`로 생성하는 것이 올바른 판단인지 확인 필요.
2. 홈으로 가기 링크(`/`)가 파트너 마케팅 사이트에서 작동할 때 미들웨어를 다시 통과하여 올바른 파트너 페이지로 리다이렉트되는지 확인 필요.

### Resolution
- Q1: WL-33.md로 저장 (오타 판단). 문경 님 확인 후 WL-35.md로 변경 가능.
- Q2: `<Link href="/">` → 동일 도메인 루트 요청 → middleware가 해당 도메인의 파트너를 재조회하여 올바르게 라우팅됨. 동작 확인됨.
