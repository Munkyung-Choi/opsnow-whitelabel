## 📝 Core Logic

`proxy.ts`에 이원화되어 있던 Admin 진입 체크(pathname 기반 + host 기반)를 **host-first 단일 진입점**으로 통합.
`isAdminHost(host)` 블록 안에서 Supabase 세션 검증을 수행하고, 마케팅 host에서 `/admin` 경로 접근 시 `/not-found`로 차단.
Playwright `auth.setup.ts`의 로그인 URL을 `admin-whitelabel.localhost:3000`으로 전환하고, 공유 auth 파일 경로를 `tests/fixtures/auth-files.ts`로 분리.

## ✋ Human Check 승인 포인트

- **호스트 우선 단일화 원칙 확정(ADR-001 반영)**: Admin 도메인이 `admin-whitelabel.opsnow.com`으로 고정됨에 따라, pathname 기반 `/admin` 체크는 더 이상 유효한 게이트가 아님. 마케팅 host에서 `/admin`은 명시적 차단 대상.
- **Admin 세션 쿠키 격리 전략**: Supabase SSR의 기본 쿠키 스코프(exact host)를 신뢰하고 별도 세팅은 추가하지 않음. MVP 단계에서 수용 가능한 위험 수준.
- **WL-53 선행 완료**: `isAdminHost` 블록 내 세션 체크는 WL-53의 RBAC 역할 검증을 Server Component 레벨에서 구현하기 위한 기반을 마련함.
