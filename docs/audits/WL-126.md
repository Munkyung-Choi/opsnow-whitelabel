### Auditor Digest — WL-126 Admin Site Builder P2

**Core Logic**
- `partner_admin`이 자신의 파트너 설정(테마·로고·파비콘)을 수정하는 Server Action(`updatePartnerTheme`)
- `withAdminAction` 7단계 체크체인 래핑, `partner_id`는 세션에서 직접 주입(form 입력 신뢰 금지)
- 파일 검증은 클라이언트(onChange 즉시) + 서버(action 진입 시) 이중 방어

**Identified Risks & Resolutions**

| 위험 | 발견 경위 | 조치 |
|------|-----------|------|
| `export type {SiteBuilderFormState}` in 'use server' 파일 → Turbopack이 runtime value로 등록 시도 → `ReferenceError` | E2E 테스트 실행 중 페이지 크래시 | `actions.ts`에서 type re-export 제거, `SiteBuilderForm.tsx`에서 직접 import |
| `partners` 테이블에 `partner_admin` UPDATE RLS 정책 부재 → 저장이 silently 0 rows 업데이트 | E2E DB 검증 단계에서 `theme_key = blue` 고정 발견 | `20260419000010` migration: `partners_partner_admin_update` 정책 추가 (Critical 트랙 Human Check 완료) |
| Next.js server action 기본 body limit 1MB → 2MB 로고 업로드 불가 | 600KB 파비콘은 통과, 3MB 로고 테스트 타임아웃으로 발견 | `next.config.ts`: `serverActions.bodySizeLimit: '3mb'` |
| `fullyParallel: true` + 동일 DB 파트너 공유 → Happy Path 3개 테스트 경합 | 첫 시도 실패, retry 성공 패턴으로 진단 | `test.describe.configure({ mode: 'serial' })` |

**Human Check 승인 포인트**
- `partners_partner_admin_update` RLS 정책 설계안 검토 및 승인 완료 (2026-04-19)
- `USING(get_my_role() = 'partner_admin' AND owner_id = auth.uid())` + `WITH CHECK(owner_id = auth.uid())`
- 소유권 이전 방지(owner_id 변경 차단) 및 크로스테넌트 차단 확인
