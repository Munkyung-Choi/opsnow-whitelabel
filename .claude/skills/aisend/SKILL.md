# aisend — 외부 AI 분석 교차검증 스킬

## 용도

외부 LLM(ChatGPT, Gemini 등)이 제시한 코드 분석·설계 제안을 **실제 코드베이스와 대조**하여 정확도를 판정한다.
"AI가 그렇다네요"를 "코드로 증명된 사실입니다"로 변환하는 것이 목적이다.

## 사용법

```
/aisend [외부 AI 분석 텍스트 또는 "현재 대화" 키워드]
```

- 외부 AI의 분석 내용을 그대로 붙여넣거나, `현재 대화`라고 입력하면 직전 컨텍스트에서 검증 대상을 추출한다.
- `--ticket WL-XXX` 옵션: 판정 결과를 해당 Jira 티켓에 댓글로 기록한다.

---

## 실행 절차

### STEP 1 — 검증 가능한 주장 목록 추출

외부 AI 분석에서 **검증 가능한 사실 주장**만 리스트로 뽑는다.

추출 기준:
- 파일·함수·컴포넌트 존재 여부 주장
- "X 패턴이 없다", "Y 로직이 중복된다" 형태의 코드 구조 주장
- "Z 라이브러리를 추가해야 한다" 형태의 의존성 주장
- 보안·성능·확장성 관련 구체적 코드 결함 지적

주관적 의견("더 나을 것 같다", "권장된다")은 목록에서 제외하고 별도 §의견 섹션으로 분류한다.

### STEP 2 — 코드베이스 교차검증

추출된 각 주장을 아래 도구로 검증한다. **"타당해 보인다"는 말로 검증을 대체하지 않는다.**

| 주장 유형 | 검증 도구 |
|----------|----------|
| 파일/함수 존재 여부 | `Glob` + `Grep` |
| 로직 중복 여부 | `Grep` (패턴 검색) |
| 의존성 존재 여부 | `package.json` Read |
| DB 컬럼·RLS 정책 | `supabase/migrations/` Grep |
| 타입 정의 | `src/types/` Read |
| 테스트 커버리지 | `tests/` Glob + Grep |

외부 AI가 접근하지 못한 파일 전체를 읽어 정보 비대칭을 활용한다.

**검색 불가 상황**: 컨텍스트 한계로 일부 항목을 이 세션에서 검증하지 못할 경우 해당 항목에 `⏳ 컨텍스트 부족 — 별도 확인 필요`를 명시하고 검증된 항목만으로 판정한다.

### STEP 3 — 판정 테이블 출력

```
## /aisend 판정 결과 — [주제 한 줄 요약]

| # | 외부 AI 주장 | 판정 | 근거 (파일:줄번호 또는 검색 결과) |
|---|------------|------|----------------------------------|
| 1 | ... | ✅ 정확 | ... |
| 2 | ... | ❌ 오진 | ... |
| 3 | ... | ⚠️ 부분 정확 | ... |
| 4 | ... | ➕ 누락 보완 | ... |
| 5 | ... | ⏳ 컨텍스트 부족 | ... |
```

판정 기호:
- **✅ 정확**: 코드에서 주장이 사실로 확인됨
- **❌ 오진**: 코드에서 주장이 사실이 아님으로 확인됨 → **역제안 필수** (아래 참조)
- **⚠️ 부분 정확**: 일부만 맞고 일부는 틀림 — 정확한 경계 명시
- **➕ 누락 보완**: 외부 AI가 언급하지 않았으나 내부 검증에서 추가 발견된 리스크/개선점
- **⏳ 컨텍스트 부족**: 이 세션에서 검증 불가 — 별도 확인 필요

**❌ 오진 시 역제안 형식**:
```
- 오진 이유: [실제 코드가 이러하므로]
- 역제안: 외부 AI의 의도를 달성하려면 [이렇게 수정/접근] 해야 한다.
```

### STEP 4 — 채택 권고

```
## 채택 권고

| 분류 | 항목 번호 | 권고 |
|------|----------|------|
| 즉시 실행 | #N | [Low 트랙 — 직접 진행 가능] |
| 티켓 발행 후 진행 | #N | [Med/High 트랙 — 별도 티켓 필요] |
| 보류·기각 | #N | [오진 또는 이미 구현됨] |
```

### STEP 5 — 추적성 기록

판정 결과를 추적 가능하게 남긴다.

- **기본**: 판정 테이블을 현재 대화에 출력한다 (Jira 댓글에 복사 가능한 형태).
- **`--ticket WL-XXX` 옵션 사용 시**: 판정 결과를 해당 Jira 티켓에 댓글로 자동 기록한다.
  - 댓글 제목: `[/aisend 판정] 외부 AI 분석 교차검증 결과 — YYYY-MM-DD`
  - 내용: STEP 3 판정 테이블 + STEP 4 채택 권고

## 공통 실수 방지

| 실수 | 방지책 |
|------|--------|
| "타당해 보인다"로 검증 대체 | 반드시 Grep/Read로 파일 확인 후 근거 명시 |
| 외부 AI가 존재한다는 파일을 확인 없이 신뢰 | 존재 여부를 Glob/Read로 직접 검증 |
| ❌ 오진 판정 후 "틀림"으로만 끝내기 | 역제안 필수 — 외부 AI의 의도를 달성하는 올바른 경로 제시 |
| 컨텍스트 부족 항목을 묵시적으로 건너뜀 | `⏳` 표시로 명시적으로 알림 |
| 판정 결과를 대화에만 남기고 티켓에 미기록 | `--ticket` 옵션 또는 복사 권고 안내 |
